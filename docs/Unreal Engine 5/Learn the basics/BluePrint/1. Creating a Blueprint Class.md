# 1. Creating a Blueprint Class
블루프린트에는 두 가지 종류가 있다

1. 블루프린트 클래스
2. 레벨 블루프린트

# Default Scene Root
최상위 부모 컴포넌트로, **모든 공간적 변환(위치, 회전, 크기)의 기준점**

## 역할
1. **계층 구조의 기준점**
    - 액터에 속한 모든 씬 컴포넌트(Static Mesh, Light, Camera 등)는
    - 반드시 디폴트 씬 루트 또는 그 자식 컴포넌트에 어태치(Attach)되어 계층 구조를 형성
    - 이 계층 구조를 통해 액터를 하나의 단위로 이동, 회전, 크기 조절
2. **필수 구성 요소**
    - 모든 액터는 반드시 하나의 루트 컴포넌트를 가져야 함
    - 블루프린트 클래스를 새로 생성하면 빈 루트 컴포넌트 역할을 하는 Default Scene Root가 기본으로 제공

## Print string
해당 노드의 Key 역할 = **변수명처럼 동일 카테고리의 메시지를 구분하는 라벨**

###  Key를 사용할 때 vs 사용하지 않을 때
- key 미사용 : 매번 **새로운 메시지로 인식되어 계속 쌓임**

```
[Hello World]
[Hello World] 
[Hello World]
```

- key 사용 : **동일한 Key를 가진 메시지는 기존 메시지를 덮어씀**

```
[Hello World] (가장 최근 시간)
```

---

> 언제 Key를 써야 할까?
Key 사용 | 	Key 미사용 |
실시간으로 업데이트되는 정보	| 일회성 이벤트 로그 |
체력, 점수, 탄약 등 게임 상태	| 에러 메시지, 디버그 정보 |
UI처럼 계속 보여줘야 하는 정보	| 임시 확인용 메시지 |

---

## 사용 노드

1. **get actor Locations**
    - 선택된 액터의 루트 컴포넌트 위치 반환 (벡터)
2. **get object Name**
    - 객체 이름 가져오기 (아웃라이너에 나오는 이름 아님)
3. **get a reference to self**
    - 자기 자신 포인터 (레퍼런스) 가져오기
4. **string : append**
    - 문자열 합치기
5. **Draw Debug Coordinate System**
    - 좌표계 그리기
    - local, world 좌표계에 따라 달라짐
    - local축과 메시 방향 맞추기
    - scale : 100 정도여야 보임

> 메쉬가 이미 루트 컴포넌트일 경우, scene 컴포넌트 추가해서 메쉬 방향 돌리기

## 블루프린트 사용하여 메쉬 움직이기
벡터 사용하여 메쉬가 점점 앞으로 가는 것처럼 만들기

### tick
1. **add actor world offset**
    - actor target에 delta add
    - 따라서 움직이는 것 처럼 보임
2. V1 - V2 한 값 normalize
    - 해당 정규화 값을 add world offset delta값에 넣으면
    - 매 틱 (프레임) 마다 조금씩 전진하도록 그려짐
3. 곱하기 노드
    - **해당 노드의 입력 핀 우클릭하면 입력값 바꿀 수 있음**

---

### 프레임 종속성
event tick에 바로 움직임을 달면 프레임에 따라 이동 결과가 달라짐

```
// 나쁜 예: 프레임 종속적
Event Tick:
    Actor 위치 += 100  // 매 틱 100씩 이동
```

- 60fps: 1초에 60틱 → 100 × 60 = 6000 이동
- 30fps: 1초에 30틱 → 100 × 30 = 3000 이동

> 프레임에 따라 속도가 달라짐

---

### 프레임 보정
움직임에 프레임 종속성 해소하기 

```
이동량 = 속도 × 시간
```

- **Delta Seconds 사용**
  - Delta Seconds = **현재 프레임의 시작 시간 - 이전 프레임의 시작 시간**

> 프레임률이 낮아지면: Delta Seconds가 커져 이동량도 같이 커짐
> - 따라서 실제 시간 동안 총 이동량을 유지

```
// 좋은 예: 프레임 독립적  
Event Tick:
    Actor 위치 += 100 * Delta Seconds  // 초당 100의 속도
```

1. event tick의 Delta Seconds 출력 핀 꺼내기
2. T = V1 - V2(normalize) * scalar (원하는 초당 움직임 단위)
3. T * Delta Seconds
4. 3번 결과값을 add actor world offset의 delta location에 꽂음

# Sequence
하나의 실행 입력을 여러 개의 순차적 실행 출력으로 분기시키는 노드

## 동작 방식
- 입력이 들어오면 Then 0 → Then 1 → Then 2 순서로 순차 실행
- 각 Then 출력은 독립된 실행 흐름을 시작하지만, **실행 순서는 보장됨**
- 병렬이 아니라 직렬 실행

## 예시
[입력 실행] → [Sequence]
  → Then 0 → [작업 A]
  → Then 1 → [작업 B]
  → Then 2 → [작업 C]

## 주의사항
1. **실행 순서는 위에서 아래로 고정**
2. **각 핀의 실행은 독립적**
   - 한 출력의 실패가 다른 출력에 영향을 주지 않음
3. **병렬 처리가 필요하면 다른 노드(예: Async Task, Multi-threading 플러그인)를 사용**
