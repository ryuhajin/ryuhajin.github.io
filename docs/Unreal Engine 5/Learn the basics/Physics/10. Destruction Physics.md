# 10. Destruction Physics
환경 파괴, 폭발, 잔해 시뮬레이션을 위한 시스템.  UE5 기준 기본 물리 엔진인 **Chaos Physics** 기반의 파괴 기능

- [UE - Chaos Destruction](https://dev.epicgames.com/documentation/ko-kr/unreal-engine/chaos-destruction-in-unreal-engine)


```
Chaos Destruction의 핵심 목적:
- 사전에 조각낸 메쉬를 물리적으로 시뮬레이션하여 현실적인 파괴 연출을 만드는 것
- 폭발/총알 충격에 따라 부서지고 흩어지는 파편 생성
- 대규모 파괴에도 안정적이며 Multi-thread 기반 시뮬레이션 제공
```

# Fracture Mode (파쇄 모드)
에디터 내에서 **Static Mesh를 Geometry Collection(파괴 가능 메쉬) 으로 변환**하고 여기에 **사전 파쇄(Pre-fracturing)** 를 수행하는 에디터 전용 모드

즉,
- **Static Mesh** = 부서지지 않는 원본 메쉬
- **Fracture Mode** = 이 메쉬를 “미리 조각” 내는 작업 과정
- **Geometry Collection** = “조각난 메쉬 + 물리 속성 + 파괴 규칙”을 포함한 파괴 가능한 오브젝트

## Geometry Collection
Chaos에서 사용되는 **파괴 가능한 메쉬 타입**

Geometry Collection은 다음을 포함한다

1. **Fractured Mesh Data**  
   - 조각난 메쉬들의 버텍스/인덱스 데이터
2. **Hierarchical Transform Tree**  
   - Chunk(조각)들이 부모-자식 구조로 정렬  
   - “어느 조각이 연결되어 있는가?”를 표현
3. **Physics Body & Collision Data**  
   - 각 조각별 물리 바디 (질량, 콜리전, 밀도 등)
4. **Clustering 정보**  
   - 여러 조각을 하나의 ‘덩어리’로 묶어서  
     충격이 더 큰 경우에만 분리되도록 설정
5. **Damage / Breaking Rules**  
   - 충격량에 따라 파괴되는 조건
   - 외부 힘(Radial Impulse 등)에 얼마나 민감한가
