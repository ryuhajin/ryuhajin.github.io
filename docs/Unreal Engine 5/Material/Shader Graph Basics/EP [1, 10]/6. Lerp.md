# EP 6 : Lerp (Linear Interpolation)
A와 B를 알파 값으로 섞기

## Lerp 기본 원리
```
Result = A × (1−α) + B × α
```

- α(알파)가 0 → 결과는 A
- α가 1 → 결과는 B
- α가 0.5 → A와 B의 중간값

### 알파에 텍스처를 넣었을 때 (Mask 역할)
알파가 단색 상수 대신 **그레이스케일 텍스처라면, 픽셀마다 값이 달라진다**

- 검정(=0)  → 그 픽셀에서 A만 출력
- 흰색 (=1)  → 그 픽셀에서 B만 출력
- 회색(=0.3 같은 중간값) → A와 B가 섞여서 출력

> 그래서 마스크 텍스처를 집어넣으면 부분별로 다르게 블렌딩 됨

---

## a,b 값에 따른 보간 방향 차이

1. a와 b의 대소 관계는 보간 결과가 a에서 b로 증가하는지 감소하는지를 결정
2. alpha는 항상 0에서 1 사이의 진행률

### a < b 인 경우 (예: lerp(0.2, 0.8, alpha))
값이 증가하는 보간
- a 라는 시작점에서 b 라는 목표점까지 점차 커짐
- 오름차순

### a > b 인 경우 (예: lerp(0.8, 0.2, alpha))
값이 감소하는 보간

- a라는 높은 시작점에서 b라는 낮은 목표점까지 점차 작아짐
- 내림차순

> lerp 함수의 a와 b는 단지 시작점과 끝점을 의미할 뿐, 크기 순서는 중요하지 않음
> - alpha가 0에서 1로 변함에 따라 결과는 항상 a에서 출발하여 b에 도착한다

```
a < b: 증가 (Fade In, 밝아지기, 나타나기)

a > b: 감소 (Fade Out, 어두워지기, 사라지기)
```

---

### 알파가 0~1 범위를 벗어났을 때
원래는 **보간(interpolation)**이지만, 값이 벗어나면 **외삽(extrapolation)**이 된다

**예: α = 2**
```
식: A × (1−2) + B × 2 = A × (−1) + B × 2 = 2B − A
```

- 즉, B를 지나서 더 나간 방향으로 색이 과장됨

**예: α = −1**
```
식: A × (1−(−1)) + B × (−1) = A × 2 − B
```

- A 방향으로 반대쪽으로 더 나감

> 결과: 색이 의도보다 더 강하게, 왜곡되거나 채도가 튀는 현상이 생김
> 의도적으로 쓰면 과포화 효과처럼 활용 가능
