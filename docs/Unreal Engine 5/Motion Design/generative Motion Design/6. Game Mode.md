# 6. Game Mode

## GameMode
게임 전체의 **룰(규칙)과 진행 흐름을 관리하는 중앙 컨트롤러**

- 어떤 Pawn/Controller/HUD/GameState를 사용할지 지정
- 플레이어 스폰 · 라운드 · 승패 규칙 등 게임 규칙을 총괄

---

## 특징  
- 서버 전용(싱글플레이에서는 호스트처럼 동작)
- 레벨/월드 단위로 존재, 레벨이 바뀌면 새로 로드될 수 있음
- 멀티플레이에서 게임 규칙을 통제하는 핵심

### 대표 역할
```
- 플레이어 스폰 규칙  
- 라운드 관리(시작·종료)  
- 점수 조건, 승패 판정  
- 게임 타이머  
- GameState/PlayerState 업데이트  
```

---

## GameMode 설정
게임 모드는 두 군데에서 설정 가능

1. **프로젝트 전체 기본값**
   - `Edit → Project Settings → Maps & Modes → Default GameMode`  
   - → 모든 맵의 기본 게임 모드
2. **개별 레벨에서 오버라이드(우선 적용)**
   - `Window → World Settings → GameMode Override`  
   - → 현재 열린 레벨에 한해서, 프로젝트 기본값 대신 다른 GameMode를 사용

즉
- 프로젝트 전역 기본 룰: Project Settings  
- 레벨별로 다른 룰/모드 사용: World Settings의 GameMode Override

---

## Game Instance
게임이 **실행 → 종료**할 때까지 살아 있는 전역 객체

- 레벨을 바꿔도 유지되며 메뉴/인게임 간 데이터 전달용으로 적합

---

## 특징  
- 프로젝트 실행 중 단 한 번 생성  
- 레벨 전환 간 데이터 유지  
- 전역 매니저(세이브·계정·설정) 위치로 활용 가능  
- 싱글/멀티 공통: 서버·클라이언트 각각 자신만의 GameInstance를 가짐  

### 대표 역할
```
- 계정/설정/선택 정보 보관  
- 메뉴 → 인게임 데이터 전달  
- 전역 SaveManager / AudioManager 등의 서비스 관리  
- 네트워크/매칭 정보 저장
```

---

## Level Blueprint
특정 레벨에만 종속되는 스크립트

- 각 레벨은 고유한 Level BP를 가지며, 해당 레벨에서만 작동하는 연출 · 트리거 로직을 담당

---

## 특징  
- 레벨 전용, 레벨 변경 시 새로 생성  
- 환경 트리거·퍼즐·컷씬 등에 사용  
- 전역 게임 규칙을 넣어서는 안 됨  

### 대표 역할
```  
- 문/문짝/엘리베이터 등 레벨 내 오브젝트 제어  
- 특정 위치 진입 시 이벤트 발생  
- 레벨 진입/퇴장 연출  
- 퍼즐 로직  
```

---

## 게임 모드 vs 게임 인스턴스 vs 레벨 블루프린트

| 구분 | 게임 모드(GameMode) | 게임 인스턴스(GameInstance) | 레벨 블루프린트(Level BP) |
|---|---|---|---|
| 생명 주기 | 레벨/월드 단위 | 게임 실행 전체 | 레벨 단위 |
| 목적 | 게임 규칙·스폰·진행 관리 | 전역 데이터·서비스 관리 | 레벨 전용 연출·트리거 |
| 레벨 이동 시 | 새로 생성(혹은 다른 GameMode로 교체) | 유지 | 새로 생성 |
| 설정 위치 | Project Settings / World Settings(GameMode Override) | Project Settings (GameInstance Class) | 각 레벨 내부 |
| 멀티플레이 | 서버 전용 | 서버/클라 각각 존재 | 서버/클라 모두 존재 |
| 사용 예 | 승패 조건, 점수, 스폰 | 계정, 옵션, 매치메이킹 | 문 열기, 컷씬, 퍼즐 |
