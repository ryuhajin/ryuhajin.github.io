# generative Motion Design

---

## Introduce
1. 현재 언리얼에서는, **메시(Mesh)**를 한 번만 분할할 수 있다
    - 후디니의 경우 절차적 방식을 사용해 이미 분할된 메시를 또 쪼개는 재귀적 분할이 가능하다
2. 따라서 기존 큐브에 완벽하게 들어맞는 8개의 더 작은 큐브를 스폰(spawn)시켜서, 마치 그것이 실제로 분할된 것처럼 보이게 만든다
3. 블루프린트에서는, 특정 조건이 충족될 때까지 이 '분할 함수(splitting function)'를 지속적으로 반복
   - while 조건문은 큐브가 분할될 수 있는 횟수의 상한선이 됨
4. 하지만 모든 큐브가 동일한 (상한선) 값을 가진다면, 모든 조각이 동일한 크기로 끝나게 되어 지루한 결과가 나옴
   - 따라서 상한선을 난수로 설정

# BP settings

## GameModeBase
게임의 규칙과 방식을 정의

### 주요 역할
- "플레이어가 몇 점을 얻으면 이기는 거지?" (승패 조건)
- "게임이 시작되면 어떤 플레이어 캐릭터(Pawn)를 생성해야 하지?" (기본 Pawn 설정)
- "플레이어가 죽으면 어떻게 되지? (리스폰 규칙)"
- "게임에 참여하는 플레이어 컨트롤러(Player Controller)는 무엇인가?"
- "맵에 HUD(화면 UI)는 무엇을 쓸까?"

### 특징

```
레벨(맵)에 오직 하나만 존재

일반적으로 레벨 안에 직접 배치하지 않고, 프로젝트 설정에서 '기본 게임 모드'로 지정
그러면 엔진이 자동으로 생성
```

## Pawn
플레이어나 AI가 조종할 수 있는 게임 내의 아바타(캐릭터 또는 물체)

### 주요 역할
- 플레이어의 입력(키보드, 마우스)을 받아 움직임을 처리
- 물리적인 충돌을 처리
- 애니메이션을 재생
- 카메라를 장착

### 특징

```
플레이어 그 자체는 아님. 플레이어가 조종하는 대상

Character 클래스는 Pawn의 가장 일반적인 자식 클래스로,
걷고 뛰는 캐릭터에 특화된 기능(예: 이동 컴포넌트)이 이미 들어있어 초보자에게 가장 많이 사용
```

### level BP
특정 맵(레벨)에서만 일어나는 사건(이벤트)을 처리하는 스크립트

> 해당 레벨에 배치된 오브젝트들 간의 1회성 또는 지역적인 상호작용을 처리

### 주요 역할
- "문에 다가가서 E 키를 누르면 문이 열리는" 트리거 이벤트
- "보스가 등장할 때 배경음악을 바꾸는" 이벤트
- "플레이어가 특정 지역에 들어오면 적을 소환하는" 트리거

### 특징

```
레벨 별로 하나씩 존재하며, 오직 그 레벨 안에서만 작동

다른 레벨에서는 재사용이 불가능

매우 쉽고 빠르게 이벤트를 만들 수 있어 초보자에게 유용하지만,
남용하면 관리하기 힘든 코드가 될 수 있어 주의가 필요
```

### 사용한 노드
1. **set view target with blend**
    - 플레이어 컨트롤러의 기능으로, 플레이어의 시점을 다른 액터(주로 카메라)로 부드럽게 전환
