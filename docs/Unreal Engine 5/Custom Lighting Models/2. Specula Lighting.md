# 2. Specula Lighting
Specular Lighting은 **표면에서 광원이 반사되는 것처럼 보이게 만드는 조명**

- **실제 반사가 아닌 Fake Reflection**
- 카메라 방향(View Direction)에 따라 변화
- 하이라이트 형태로 표현됨

---

## Specular의 핵심
Specular는 다음 관계를 기반으로 만들어짐

```
빛 → 표면 → 반사 → 카메라
```

```
1. Light Direction (광원)
2. Normal (표면)
3. View Direction (카메라 벡터)
```

이 3개 벡터가 핵심 요소이다

---

## Unreal Directional Light의 방향
- 실제 빛은 위(하늘)에서 아래(지면)로 내려옴
  - 즉, 벡터는 `(0, 0, -1)` 방향
- 하지만 계산의 편의성을 위해 광원의 방향을 **반전(Negate)**시켜서 표면에서 태양을 바라보는 방향으로 바꿈

```
표면 노멀은 보통 위를 향하는 (0, 0, 1)

만약 빛이 내려오는 방향 그대로 계산하면 내적 결과가 -1
→ 따라서 riection light를 반전 → 표면 노멀이 정면으로 마주칠 때 최대 밝기 1 출력 
```

> SkyAtmosphereLightDirection을 사용할 때 결과가 이상하다면 Multiply -1 해보기

> 언리얼의 대기 시스템 노드들은 종종 '빛이 나가는 방향'을 출력하기 때문에, 라이팅 연산을 위해서는 이를 반전시켜 '빛이 오는 방향'으로 바꿔줘야 할 때가 많음

---

## Normalize vs Saturate 차이점
두 노드 모두 결과에 1이라는 숫자가 관여하지만, 다루는 데이터의 성격이 다르다

### Normalize (정규화)
- **대상**: 벡터(Vector)
- **목적**: 벡터의 방향은 유지하되, 길이(Magnitude)만 1로 만든다
- **이유**:

```
내적 결과가 순수하게 두 벡터 사이의 각도 값만 나오게 하려면 두 벡터의 길이가 반드시 1이어야 함

- 길이가 1이 아니면 내적 결과에 벡터의 길이가 곱해져 조명 값이 너무 밝아지거나 어두워진다
```

---

### Saturate (포화)
- **대상**: 스칼라(Scalar) 또는 각 채널의 값
- **목적**: 값을 `[0, 1]` 범위로 제한(Clamp)
- **이유**:

```
모니터는 0(검정)보다 어둡거나 1(흰색)보다 밝은 색을 표현할 수 없음

- 특히 음수 조명 값은 다른 연산과 합쳐질 때 원치 않는 결과 (검은 점 등)를 낼 수 있음
- 따라서 0 이하를 자름
```

# 사용 노드
## CustomReflectionVector vs ReflectionVectorWS
1. **CustomReflectionVector**
    - HLSL의 `reflect(I, N)` 함수를 호출
    - **특징** : ReflectionVectorWS 노드가 카메라의 반사를 주로 다룬다면, CustomReflectionVector는 입력 핀을 통해 **내가 원하는 벡터(예: 태양광)**를 임의로 반사시킬 때 사용
2. **ReflectionVectorWS**
    - 카메라(View) 벡터를 표면 노멀에 반사시킨 결과를 출력하는 특수 목적 노드
    - **특징**: 별도의 입력 핀이 없거나 매우 제한적이며, 호출하는 즉시 "카메라 시선"을 기준으로 계산

| 노드 | 입력값 | 계산식 | 주요 용도 |
|------|--------|--------|------------|
| CustomReflectionVector | 사용자 정의 벡터 + 노멀 | `reflect(-UserVector, Normal)` | 비표준 반사, 스타일화된 효과, 특수 VFX |
| ReflectionVectorWS | 뷰 벡터 (CameraVectorWS) + 노멀 | `reflect(-View, Normal)` | 표준 환경 반사, 큐브맵 샘플링, 리플렉션 캡처 |

---

## CameraDirectionVector vs CameraVectorWS
1. **CameraDirectionVector**
    - **카메라가 바라보는 정면 방향(Forward Vector)**
    - **특징**: 픽셀 위치와 상관없이 카메라의 렌즈가 향하는 방향 그 자체를 의미
2. **CameraVectorWS**
    - 픽셀(표면)에서 카메라(눈)를 향하는 방향 벡터
    - **특징**: 픽셀의 위치마다 방향이 다름

| 노드 | 공간(Space) | 픽셀별 변화 | 의미 | 주요 용도 |
|------|-------------|--------------|------|------------|
| CameraDirectionVector | 월드 공간 | 불변 (Constant) | 카메라가 바라보는 방향 (카메라의 정면) | 카메라 방향 자체가 필요한 효과 (예: 비, 눈 방향) |
| CameraVectorWS | 월드 공간 | 가변 (Varying) | 픽셀 위치에서 카메라를 향하는 방향 (시선 벡터) | 스페큘러 하이라이트, 림 라이트, 반사 계산 |

## Phong specular
> 핵심 : 빛이 반사된 방향(R)이 카메라 방향(V)과 얼마나 비슷한가?

```c++
R = reflect(-L, N)
spec = dot(R, V)
```
- 빛이 표면에서 튕겨서 나가는 방향 R을 계산
- 그 방향이 카메라와 일치하면 하이라이트 발생

---

## Blinn specular
> 핵심 : 노멀(N)이 빛(L)과 카메라(V)의 ‘중간 방향’을 얼마나 향하고 있는가?

```
       V (카메라)
        ↑
        |
        |   ← H (Half Vector)
        |
        ↓
       L (빛)
```

```c++
H = normalize(-V + L)
spec = dot(N, H)
```

- 카메라 벡터 V와 광원 L 사이의 중간 방향(H) 생성
- 표면이 그 방향을 향하면 하이라이트 발생
