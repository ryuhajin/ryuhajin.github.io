# Game optimization

# 게임 최적화 기초: 원칙과 전략 가이드
게임 개발에서 최적화는 단순히 '사양을 낮추는 것'이 아니라, **사용자의 상호작용(Interactivity)을 보존하는 핵심 작업**

---

## 1. 최적화의 본질: 왜 해야 하는가?
영화와 게임의 결정적인 차이는 **상호작용성**에 있다

* **영화:** 시각적 품질과 스토리텔링이 최우선. 렌더링 시간이 길어져도 결과물만 완벽하면 됨
* **게임:** 플레이어가 실시간으로 조작. 프레임이 낮아지면 조작감이 떨어지고 몰입감이 깨짐

> 결론 : 게임에서 성능(Performance)은 그래픽 퀄리티보다 우선순위가 높다. 아무리 예뻐도 느리면 좋은 게임이 될 수 없음

---

## 2. 최적화의 책임: 누가 해야 하는가?
최적화는 프로그래머만의 영역이 아니다. 특히 **아티스트의 참여**가 필수적

| 구분 | 역할 및 장점 |
| :--- | :--- |
| **아티스트** | 아트의 **의도(Intent)**를 가장 잘 이해함. 시각적 메시지를 유지하면서 무엇을 뺄지 결정할 수 있음 |
| **엔지니어** | 기술적인 병목을 찾아내지만, 아트의 중요도를 모르기에 **무차별적으로 리소스를 삭제**할 위험이 있음 |

> 핵심: 아티스트가 직접 최적화 방법을 배울 때, 비주얼 퀄리티와 성능 사이의 최적의 균형을 찾을 수 있다

---

## 3. 성능 측정의 기준: FPS vs Frame Time
성능을 개선하려면 먼저 정확하게 측정해야 한다

### 프레임 속도와 프레임 시간 비교
**Frame Time 사용 권장**

| 측정 항목 | 정의 | 특징 |
| :--- | :--- | :--- |
| **Frame Rate (FPS)** | 초당 렌더링되는 프레임 수 | 일반적인 지표이나, 성능 변화가 **비선형적**임 |
| **Frame Time (ms)** | 한 프레임을 그리는 데 걸리는 시간 | 성능 변화가 **선형적**이며, 실제 계산량 파악에 유리함 |

### 왜 Frame Time을 써야 할까?
똑같은 5 FPS 개선이라도 실제 연산량의 차이는 천차만별

### 수식으로 이해하기
* **프레임 타임(ms) = 1000 / 프레임 레이트(FPS)**

---

```
[저속 구간]

25 FPS	= 1000 ÷ 25 =	40.0 ms
30 FPS	= 1000 ÷ 30 =	33.3 ms
        = 프레임 타임 6.7 ms 감소

저속 구간 (25→30 FPS)
 - 한 문제 푸는 데 40초 걸리던 사람이 33초로 줄었어요! 7초나 단축됐네요! → 엄청난 개선
```
```
[고속 구간]

120 FPS	 = 1000 ÷ 120 =	8.33 ms
125 FPS	 = 1000 ÷ 125 =	8.00 ms
         = 프레임 타임 0.33 ms 감소

고속 구간 (120→125 FPS)
 - 한 문제 푸는 데 8.33초 걸리던 사람이 8초로 줄었어요. 0.33초 단축... 거의 느껴지지도 않네요
```

---

### 결론
FPS 수치만 보면 5 FPS로 같아 보이지만, 첫번째 케이스가 훨씬 큰 최적화 성과를 낸 것

> 따라서 **밀리초(ms)** 단위로 추적해야 정확한 분석이 가능

> 최적화의 목표는 "FPS 숫자 올리기"가 아니라, "프레임 타임 줄이기"이다!

---

## 4. 최적화의 3대 황금 원칙

### (1) "선(先) 분석, 후(後) 최적화"
* **추측하지 말자**  : 어디가 느린지 데이터로 확인한 뒤 그 부분에 집중하기
  * 원인을 모르고 설정을 건드리면, 성능은 안 오르고 그래픽만 망가질 수 있음

### (2) "자산(Asset)과 레벨(Level)의 조화"
* **에셋 최적화:**
    - 개별 모델의 폴리곤 수, 텍스처 압축, LOD 설정 등 (제작 단계에서 수행)
* **레벨 최적화:**
    - 최적화된 에셋이라도 1만 개를 깔면 느려짐
    - 드로우 콜(Draw Calls), 배치 처리(Batching) 등 배치 방식을 최적화해야 함

### (3) "프로젝트 전 기간에 걸친 프로세스"
1. **초기:** 타겟 하드웨어 사양 파악 및 성능 예산(Budget) 수립
2. **중기:** 제작 중 실시간 메트릭 확인 및 예산 준수
3. **말기:** 병목 지점에 대한 정밀 타격 최적화

> 마지막 날에 몰아서 하는 최적화는 실패할 확률이 높다
