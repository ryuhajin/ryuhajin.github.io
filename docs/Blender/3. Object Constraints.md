## Object vs Mesh
### Object
씬 안에서 **하나의 좌표계와 변환값을 가지는 인스턴스**

```md
Cube Object
 ├─ Location / Rotation / Scale
 └─ Mesh Data (CubeMesh)
```

### Object가 가진 것
- 위치(Location)
- 회전(Rotation)
- 스케일(Scale)
- 어떤 데이터를 쓸지에 대한 참조


---

### Mesh
**형태를 정의**하는 순수 데이터 덩어리

### Mesh Data가 가진 것
- 버텍스
- 엣지
- 페이스
- 노멀
- UV

```md
Mesh Data는

1. 위치 개념이 없다
2. 씬에 “존재”하지 않는다
3. 혼자서는 보이지 않는다

👉 반드시 Object를 통해서만 씬에 나타남
```

## 정리

| 구분 | Modifier | Constraint |
|---|---|---|
| 대상 | 메쉬(형태) | 오브젝트 / 본 Transform |
| 핵심 | 지오메트리 계산 | 관계·종속 기반 Transform 계산 |
| Apply | 계산 결과를 메쉬 데이터로 굳힘 | 계산된 현재 Transform 값을 객체/본에 복사한 뒤 제약 제거 |
| Apply 성격 | 구조적 (지오메트리 확정) | 일회성 (현재 상태 고정) |
| 규칙 유지 | Apply 후에도 결과 구조 유지 | Apply 시 규칙(관계) 즉시 소멸 |
| 용도 | 모델링 / 절차적 형태 생성 | 애니메이션 / 리깅 / 자동화 |

> Constraint Apply 시 규칙의 현재 계산값을 숫자로 복사함
> - 따라서 관계 로직은 사라지고, 그 순간의 Transform만 남음

---

## Constraint란 무엇인가?
**Constraint(제약조건)**는 오브젝트의 **Transform(위치, 회전, 스케일)**을 다른 오브젝트나 규칙에 **종속되도록 계산하는 시스템**이다

```md
Constraint를 사용해 기사의 손에 검을 고정 하거나
테니스 선수의 눈이 공을 향하도록 만들 수 있다!
```

> Constraint는 오브젝트(Object) 또는 본(Bone)에 적용된다

## 핵심 특징
- 메쉬(형태)는 변경하지 않는다
- 오브젝트의 **위치/회전/스케일**만 계산한다
- **관계(Relationship)를 정의**하는 로직이다

> 즉, Constraint는 오브젝트 간의 관계와 규칙을 계산하는 도구

---

## 관계 기반 계산
- 이 오브젝트는 **무엇을 기준으로** 움직여야 하는가?
- 어느 정도까지 **따르거나 제한**되어야 하는가?

그래서 모든 Constraint는 기본적으로
- **‘나(this)’ + ‘타겟(target)’ + ‘규칙(rule)’** 구조를 가짐

### 중요
- Constraint는 **매 프레임 계산**된다
- 결과는 항상 계산값이며, 고정된 값이 아니다

---

## 눈 모양 아이콘
제약 조건을 **활성화하거나 비활성화**할 수 있음

- 제약 조건을 비활성화하면 해당 효과는 사라지지만 설정은 그대로 유지
- 제약 조건을 비활성화하는 또 다른 방법은 영향력 값을 0으로 설정하는 것 (애니메이션에서 활용)

---

## target
제약 조건이 참조해야 하는 객체

- 기본적으로 **대상의 원점 위치를 사용**한다

---

## influence
기본값은 1이며, **제약 조건이 객체의 원래 위치를 완전히 재정의**한다는 의미

- 0~1 사이로 설정하면 보간된 결과가 나타남

---

## Track the Camera to Objects
카메라가 오브젝트 쫒게하기

1. 카메라 선택
2. 제약조건 추가 - Track To 선택
3. target 선택 - 트래킹할 오브젝트 선택
4. 해당 오브젝트를 움직이면 카메라가 따라감
