# 2. Object Modifier
- [Modifier 메뉴얼 링크](https://docs.blender.org/manual/en/latest/modeling/modifiers/introduction.html)

## Modifier란 무엇인가?
**Modifier(모디파이어)**는 메쉬 오브젝트의 지오메트리(버텍스, 엣지, 페이스)를  
직접 편집하지 않고 **규칙에 따라 자동으로 변형하는 시스템**이다

## 핵심 특징
- 메쉬를 **비파괴적(Non-Destructive) 으로 수정**한다
- **원본 메쉬 데이터는 유지**된다
- 결과는 **계산된 상태로 화면에 표시**된다

즉, Modifier는  
> 메쉬를 직접 깎는 도구가 아니라 메쉬 위에 얹는 계산 규칙 레이어

---

## Modifier Stack (스택) 구조
Modifier는 **위에서 아래 순서대로 계산되는 스택 구조**를 가짐

```md
원본 메쉬
→ Modifier 1
→ Modifier 2
→ Modifier 3
= 최종 결과
```

- 위쪽 Modifier의 출력이
- 아래쪽 Modifier의 입력이 된다
- **순서를 바꾸면 결과도 바뀔 수 있다**

---

## Modifier 분류
1. **Generate**
    - 형태를 생성/추가 (Array, Mirror, Solidify 등)
2. **Deform**
   - 기존 형태를 변형 (Simple Deform, Lattice 등)
3. **Normals**
    - 쉐이딩/노멀 계산 (Weighted Normal 등)
4. **Subdivision**
    - 면을 분할해 부드럽게 만듦
5. **Physics/Simulation**
    - 물리 기반 계산

---

# Modifier 규칙
## Modifier는 결과를 보여줄 뿐이다
Modifier는 보이는 결과를 바꾸지만 Edit Mode의 원본 지오메트리는 그대로일 수 있음

따라서
- Edit Mode에서 버텍스를 움직이면
- Modifier가 적용되기 전 상태를 기준으로 편집된다

**이를 보완하기 위해**
- Modifier에는 **Edit Mode 표시, On Cage 같은 옵션이 존재**한다

---

## Modifier 적용 (Apply)
Modifier의 **Apply**는 다음을 의미

> 지금까지 계산된 결과를 실제 메쉬 지오메트리로 굳힌다

1. **Apply를 하면**
   - 비파괴성이 사라진다
   - 버텍스 수가 실제로 증가/변형된다
   - Modifier는 스택에서 제거된다
2. **따라서 일반적인 원칙은**
   - 작업 중에는 Apply를 최대한 미루고
   - 최종 단계에서만 신중하게 Apply한다

---

# Modifier 패널 헤더
이 Modifier가 언제, 어디에, 어떤 방식으로 계산되는지 제어

> 아이콘을 눌러 ON/OFF 할 수 있음

## Modifier 헤더 개념
패널 헤더는 **Modifier 자체의 로직 조건**을 관리

- 이 Modifier를 **계산할 것인가?**
- **어디에서 보이게 할 것인가?**
- **어떤 상태에서 유효한가?**를 결정한다

---

## 패널 헤더 주요 로직
![](/images/Blender_modeling_modifiers_panel-layout.png){: width="60%" height="60%"}

첫번째 아이콘부터 순서대로 정리

## 케이지
편집 기준을 **원본이 아니라, Modifier 결과 형태에 맞춰 보정**할 것인가?

- Subdivision, Mirror 등에서 중요
- ‘보이는 형태를 기준으로 편집’ 가능하게 함

---

## 편집모드
**Edit Mode에서도 Modifier 결과를 참고**할 것인가?

- 원본 편집 + 결과 미리보기
- 비파괴 구조를 유지하면서 형태 파악 가능

---

## Realtime 표시
**3D Viewport에서 이 Modifier를 계산**할 것인가?

- 뷰포트와 렌더를 분리 제어 가능
- 테스트용 Modifier를 뷰포트에서 제외할 수 있음

---

## Render 표시
**render에서 이 Modifier를 계산**할 것인가?

- 테스트용 Modifier를 렌더러에서 제외할 수 있음

---

## Apply
**Ctrl + A 키로 사용**

- 계산 규칙을 종료하고, 결과를 실제 메쉬 데이터로 확정
- 되돌리기 어려움
- 파이프라인 분기 지점

---

## Remove (X 버튼)
- Modifier 규칙 자체를 제거
- 원본 메쉬에는 영향 없음

---

# 정리

| 구분    | Edit Mode   | Cage Mode      | Spline(Curve) Edit Mode |
| ----- | ----------- | -------------- | ----------------------- |
| 편집 대상 | 메쉬 원본       | 메쉬 원본(보정됨)     | 곡선 데이터                  |
| 기준 좌표 | Modifier 이전 | Modifier 이후 형태 | 수학적 곡선                  |
| 실제 수정 | 지오메트리       | 지오메트리          | 곡선 파라미터                 |
| 목적    | 구조 수정       | 결과 기준 수정       | 경로/형태 규칙                |


1. **Edit Mode**
    - → 원본 메쉬를 직접 수정하는 유일한 모드
2. **Cage Mode**
    - → Modifier 결과 형태를 기준으로 편집 입력을 보정하는 옵션
3. **Spline(Curve) Edit Mode**
    - → 메쉬가 아니라 곡선의 수학적 정의 (곡선 규칙 함수 종류 선택)를 편집하는 모드
