# 2. Procedural Shader Nodes
주로 쓰이는 노드 정리

---

# Material Output
## Render target (렌더 엔진 설정)
- All: EEVEE와 Cycles 모두에서 이 Output을 사용
- EEVEE: EEVEE에서만 이 Output을 사용
- Cycles: Cycles에서만 이 Output을 사용

> 한 머티리얼 안에 Material Output을 여러 개 두고 엔진별로 다른 셰이더 트리를 쓰게 하기 위함

### EEVEE
실시간 렌더러

- 빠르게 그럴듯한 결과를 즉시 확인하는 데 최적
- 매우 정확한 광학 효과에서는 한계
  
---

### Cycles
프로덕션용 렌더러

- 물리 기반 패스 트레이서(path tracer)로 광선을 추적
- 전역조명/반사/굴절 등을 더 물리적으로 계산

```md
속도/반복/실시간 미리보기가 중요하면 → EEVEE

최종 품질/물리적 정확도가 중요하면 → Cycles
```

---

## Surface
**물체 표면**에서의 빛 반응(반사/확산/굴절/광택 등)을 출력

- 보통 **Principled BSDF 같은 ‘Shader(셰이더) 출력’**이 연결

### 연결 과정 (대표 패턴)
1. Principled BSDF(또는 Glass/Emission/Toon 등)에서 BSDF(Shader 출력) 생성
2. 필요하면 Mix Shader, Add Shader 등으로 여러 셰이더를 조합
3. 최종 Shader 출력을 Material Output의 Surface에 연결

> 핵심: Surface는 색(Color)이 아니라 Shader(빛 반응 정의)를 받음

---

## Volume
**물체 내부** (또는 월드) 의 부피에서의 빛 반응을 출력

- 연기/안개/젤리 같은 **흡수(Absorption), 산란(Scattering) 성질**이 여기에 해당

### 연결 과정 (대표 패턴)
1. Principled Volume 또는 Volume Scatter, Volume Absorption 같은 Volume 셰이더 노드를 사용
2. 필요하면 값(밀도, 색)을 Noise/Musgrave 등으로 변조
3. 최종 Volume Shader를 Material Output의 Volume에 연결

> 표면(Surface)과 부피(Volume)는 동시에 쓰일 수 있다. 예: 유리 표면 + 내부 안개

---

## Displacement
지오메트리를 **변형하는 출력 (높이 기반 변형)**

- Displacement 입력이 별도로 있으며, 범프처럼 빛만 속이는 것이 아니라 **실제 변형(또는 설정에 따라 범프/디스플레이스 혼합)**으로 이어질 수 있다

### 연결 과정 (대표 패턴)
1. 높이 데이터 만들기: Noise/Voronoi/이미지 Height 맵 등(대개 흑백 값)
2. Displacement 노드(Shader > Vector > Displacement)를 사용해 Height / Scale / Midlevel 등을 조절
3. 그 출력(Vector)을 Material Output의 Displacement에 연결

```md
Displacement가 진짜로 지오메트리를 움직이려면 보통 Cycles에서 의미가 큼

- 머티리얼 설정에서 Displacement 모드(예: Displacement and Bump)와
- 메쉬 서브디비전(또는 Adaptive Subdivision) 같은 조건이 필요하다
```

---

## Thickness (Blender 4.2 + EEVEE Next에서 중요해진 소켓)
EEVEE(특히 EEVEE Next)에서 **물체의 두께(내부 경로 길이)를 근사/지정하는 입력**

- 연결하지 않으면 Blender가 기본 두께를 추정하고, 연결한 값을 오브젝트 공간 두께로 사용한다

### 연결 과정 (대표 패턴)
1. **단순 고정값**
    - Value 노드(예: 0.0 또는 원하는 두께)
    - Material Output → Thickness에 연결
2. **텍스처/맵 기반 (두께 맵)**
    - 이미지/절차적 값으로 thickness 마스크 생성
    - Map Range/Math로 스케일 맞춤
    - Thickness에 연결

```md
[언제 쓰는거야?]

1. 굴절/투과/SSS(서브서피스) 등에서 물체가 얼마나 두꺼운지가 결과에 영향을 줄 때
2. EEVEE에서 내부 구조를 완전한 레이 트레이싱으로 계산하기 어려운 부분을 근사할 때
```

---

# Principled BSDF
Blender의 표준 PBR(물리 기반) 표면 셰이더

- 표면이 빛에 어떻게 반응하는지를 이 노드에서 대부분 처리
- 해당 노드의 BSDF 출력이 Material Output → Surface로 들어간다

> base color, roughness, metalic은 설명 패스

---

## IOR
**빛이 매질 안에서 얼마나 굴절 (방향 변화)되는가** 를 나타내는 값

### 체감 되는 곳
- 굴절 각도(Transmission 있을 때)
- 프레넬(Fresnel) 반사량(특히 가장자리 반사) 변화

### 값이 출력에 주는 영향
1. **IOR ↑**
    - 투명 재질에서는 굴절이 더 강해져 **내부가 더 꺾여 보임**
    - 반사에서는 가장자리(글랜싱 각) 반사가 더 강해지는 경향 (프레넬 효과가 커짐)
2. **IOR ↓ (1.0에 가까움)**
    - 굴절이 약해지고 반사도 약해짐

---

## Normal
보통 Bump 또는 Normal Map 노드를 입력

- 절차적 연결 흐름
    - Noise/Voronoi (흑백 height) → Bump(Height) → Normal
- Normal은 방향 벡터라서 **중간 변환 노드가 필요**

---

## Subsurface (SSS, 서브서피스 스캐터링)
표면 아래로 들어간 빛이 내부에서 여러 번 산란하고 다시 나오는 현상

- 피부, 왁스, 우유, 대리석 같은 재질에 쓰인다

```md
SSS는 보통 추가 파라미터가 함께 있음 (버전에 따라 이름/구성이 조금 다름)

- Radius / Scale (또는 비슷한 형태): RGB별로 내부에서 얼마나 멀리 퍼지는지 (피부 톤 튜닝의 핵심)
```

### 값이 출력에 주는 영향
1. **Subsurface(Weight/Amount) ↑**
    - 그림자 경계가 부드러워지고, 하이라이트와 명암이 “살”처럼 퍼지는 느낌
    - 디테일이 약간 뭉개져 보임 (내부 확산 때문)
2. **Subsurface ↓ (0에 가까움)**
    - 일반 불투명 표면 (확산+반사) 으로 돌아감

---

## Emission
스스로 빛나는 표면 (네온, LED)

- 발광은 조명처럼 동작할 수도 있고(특히 Cycles), EEVEE에서는 설정/샘플에 따라 영향이 달라질 수 있음

---

## Transmission
빛이 표면을 통과하는 정도 (0 = 불투명, 1 = 투명)

### 함께 봐야 하는 것
- **IOR**: 굴절률(유리/물/플라스틱 느낌 결정)
- **Specular**(또는 그에 해당하는 반사 조절): 표면 반사 강도에 관여

> Transmission을 올리면 표면 모델이 투명 재질 모드로 들어간다
> - 그 상태에서 IOR과 Roughness가 최종 출력을 크게 좌우한다

---

## Coat
Coat는 기본 재질 위에 얇고 매끈한 투명 코팅 레이어가 한 겹 더 있는 것을 모델링 함

- 자동차 클리어코트, 바니시(니스), 코팅된 플라스틱 등
- 다른 레이어 위에 추가되는 흰색 스페큘러 레이어

### 값이 출력에 주는 영향
1. **Coat Weight ↑**
    - 위에 얇은 반사층이 생겨서 하이라이트가 한 겹 더 생김
    - 젖은 느낌 / 코팅된 광택이 올라옴
2. **Coat Roughness ↑**
    - 코팅 하이라이트가 퍼지면서 무광 코팅 느낌
3. **Coat Normal** (필요시 사용)
    - 베이스 노멀과 별개로 코팅층만 매끈하게 (혹은 미세 스크래치만) 만들 수 있음
    - 예: 베이스는 거친 플라스틱인데 코팅층은 거의 매끈

---

## Sheen
섬유(fibers)가 만들어내는 부드러운 반사를 근사

- 천 재질에서 가장자리 부근에 벨벳 같은 반사 표현

### 값이 출력에 주는 영향
1. **Sheen Weight ↑**
    - 정면보다 가장자리 쪽에서 부드러운 빛 띠가 생김
    - 천, 벨벳, 미세 먼지 낀 표면 느낌에 유효
2. **Sheen Roughness ↑**
    - 가장자리 반사가 더 퍼지고 소프트해짐

---

## Thin Film (박막 간섭)
표면 위에 나노~마이크로미터급 아주 얇은 막이 있을 때 발생하는 **간섭 색(무지개빛)**을 시뮬레이션

- **Film Thickness(두께, 보통 nm 단위)** : 두께에 따라 색 띠 패턴이 변함
- **Film IOR (막의 IOR)**: 간섭 강도/색 분포에 영향

### 값이 출력에 주는 영향
1. **Thickness 변화 (특히 10~1000nm 대역에서)**
    - 무지개색 띠가 바뀌고 비누방울/오일막 느낌이 달라짐 (두께가 색을 결정)
2. **Film IOR 변화**
    - 같은 두께라도 색이 달라짐 (막 자체의 광학 성질 변화)

---

# Noise Texture
기본 노이즈 함수이며 그 위에 여러 단계(옥타브)의 노이즈를 겹친 프랙탈 구조

- 연속적인 **난수 필드**(smooth random field)를 생성
- **공간 좌표 → 값**으로 매핑하는 수학 함수
- **노이즈는 값 분포**다
- 그 값을 우리가 **색, 거칠기, 높이, 마스크로 해석**하는 것뿐이다!

> 즉, 이 위치(x, y, z)에서의 랜덤한 값은 얼마인가? 를 묻는 노드

---

## 출력 소켓
1. **Fac (Factor)**
    - 범위 : 0 ~ 1
    - 의미 : 이 위치의 노이즈 값
2. **Color**
    - Fac 값을 RGB로 변환한 것
    - 디버깅이나 색패턴용

> 마스크, 수치 제어는 Fac 사용

---

## Noise Texture Type
여러 레이어 (옥타브) 겹치는 방식 제어

### fBM (Fractal Brownian Motion)
각 옥타브 값을 더해서(add) 쌓는 방식

- 비교적 균일하고 방향성/지역 편차가 적다 (전체적으로 고르게 거칠어짐)
- 범용 (얼룩, 표면 거칠기, 기본 마스크)

---

### Multifractal
옥타브 값을 곱해서(multiply) 결합

- 위치에 따라 강도가 달라지는 불균일(uneven) 패턴
- 지형 느낌, 얼룩이 군데군데 몰리는 패턴 (침식과 같은 지역성에 유리)

---

### Ridged Multifractal
노이즈를 절댓값 처리하고 (계곡을 만들고) 뒤집는 방식으로 능선(ridge)을 만든다

- 선명한 능선/크랙/주름 같은 구조
- 갈라짐, 주름, 바위 결, 산 능선
- 대비가 강해지기 쉬워서 보통 Clamp로 범위를 먼저 정리하고 쓰는 편이 안정적

---

### Hybrid Multifractal
옥타브를 덧셈+곱셈을 섞어 결합

- 평평한 영역에서 갑자기 산이 솟는 듯한 피크/골짜기 + 지역별 거칠기 변화
- 바위/산악 지형, 큰 덩어리(저주파) + 날카로운 봉우리(고주파) 에 쓰임

---

### Hetero Terrain
Hybrid Multifractal과 비슷하지만 하천/수로 같은 채널 느낌이 더 나는 지형

- 침식된 지형, 강줄기/골짜기 패턴 베이스

---

### Normalize 체크
켜면 출력이 0~1 범위 안에 유지되도록 정규화

- 옥타브를 더하다 보니 Detail이 커지면 이론상 범위가 커질 수 있음
- 정규화로 항상 다루기 쉬운 범위로 맞춘다

---

## 입력 소켓
## Scale
노이즈의 공간 주파수

- 얼마나 자주 변화하나

### 값이 출력에 주는 영향
1. **Scale ↓ (1~3)**
    - 큰 덩어리
    - 얼룩, 지형
2. **Scale ↑ (20~100)**
    - 잔무늬
    - 표면 미세 거칠기

---

## Detail
옥타브 추가량 (고주파 성분 추가). 프랙탈의 노이즈 개념

### 값이 출력에 주는 영향
1. **Detail = 0**
    - 부드러운 단순 노이즈
2. **Detail ↑**
    - 잔 디테일이 겹쳐짐
    - 자연스러워지지만 노이즈도 증가

---

## Lacunarity
각 디테일 레이어의 주파수 증가 비율

### 값이 출력에 주는 영향
1. **Lacunarity = 2.0 (기본)**
    - 각 단계가 2배로 촘촘
    - 자연계에서 가장 흔한 비율
    - 안정적
2. **Lacunarity ↑ (3.0 ~)**
    - 디테일이 급격히 작아짐
    - 고주파가 빨리 등장
    - 표면이 복잡/거칠어짐
3. **Lacunarity ↓ (1.2 ~ 1.6)**
    - 각 단계가 비슷한 크기
    - 덩어리감 유지
    - 부드럽고 유기적

---

## Roughness (Noise의 Roughness)
디테일 레이어가 얼마나 거칠게 섞이는지

### 값이 출력에 주는 영향
1. **Roughness ↓**
    - 부드럽고 흐릿한 노이즈
2. **Roughness ↑**
    - 날카롭고 대비가 강해짐

> 패턴의 성격을 바꾸는 속성

---

## Distortion
노이즈 좌표 자체를 뒤틀기

### 값이 출력에 주는 영향
1. **Distortion = 0**
    - 깨끗한 노이즈
2. **Distortion ↑**
    - 소용돌이, 찌그러짐
    - 유기적인 왜곡

> 연기, 얼룩, 액체 느낌에 자주 사용

---

## 대표적인 사용 패턴
- 색 얼룩 (돌, 흙, 페인트)

```md
Noise(Fac)
 → ColorRamp (색 2~3개 그라데이션)
   → Base Color
```

- roughness 변조 (불규칙적인 표면 만들기)

```md
Noise(Fac)
 → Map Range OR ColorRamp
   → Roughness
```

- 마스크 생성

```md
Noise(Fac)
 → ColorRamp (0 아니면 1)
   → Mix Factor
```

---

## 정리
- Noise는 색이 아니라 값이다
- Scale은 크기, Detail은 복잡도
- Lacunarity는 복잡도의 촘촘함, Roughness는 다음 복잡도의 영향력

---

# Texture Coordinate
표면의 **한 점 (픽셀)이 어디에 있는지**를 값(Vector)으로 알려주는 노드

- 즉 픽셀의 x, y 값 (좌표)를 알려주는 노드

---

![](/images/blender-texcoord.jpg){: width="60%" height="60%"}


## 출력 소켓

## Generated
가장 많이 쓰이는 기본 값

- 오브젝트의 바운딩 박스(0~1) 기준 좌표
- UV 없이도 자동으로 생성됨
- 현재 생긴 모양에 종속됨

### 특징
- 오브젝트 크기에 따라 패턴 스케일이 변함
- UV 언랩 불필요

---

## UV
UV 언랩으로 정의한 2D 좌표 (사용자가 직접 펼쳐서 정의한 2D 좌표)

### 특징
- 오브젝트 크기/변형에 영향 없음
- 이미지 텍스처와 정확히 대응
- UV 작업이 필요

---

## Object
오브젝트 공간 기준 좌표

- 특정 오브젝트의 로컬 좌표계 기준
- 오브젝트를 기준으로 좌표를 계산

### 특징
- 오브젝트를 **이동/회전해도 패턴 유지**
- 스케일에는 영향 받을 수 있음

---

## World
씬 전체 절대 좌표

- 오브젝트가 움직여도 패턴이 월드 공간에 고정
- 월드 기준 정렬 효과

---

## Normal / Reflection / Camera (고급/특수)
1. **Normal**
    - 법선 방향 기반 좌표 (엣지/각도 마스크)
2. **Reflection**
    - 반사 방향 (특수 효과)
3. **Camera**
    - 카메라 기준 투영 (프로젝션)

# Mapping
좌표(Vector)를 이동 · 회전 · 스케일 변환하는 노드

> 텍스처 자체를 바꾸는 게 아니라, 텍스처를 읽는 좌표계를 바꾼다!

```md
[mapping이 필요한 이유?]
mapping 노드는 x,y,z 좌표 자체를 바꾼다

- 같은 noise라도 좌표를 바꾸면 완전히 다른 패턴처럼 보인다
```

## 기본 사용 패턴
Mapping은 Texture Coordinate와 노이즈 패턴 노드 사이의 필터이다

```md
Texture Coordinate
 → Mapping
   → Noise / Voronoi / Image
```

---

## Mapping Type
Type을 통해 이 Vector를 **점**으로 볼지, **방향**으로 볼지, **표면 법선**으로 볼지 결정한다

## 좌표 (point)
입력 벡터를 **공간상의 위치 좌표**로 해석
    
- Location / Rotation / Scale 전부 적용
- 완전한 아핀 변환(Affine Transform)

### 사용
- 절차적 텍스처 전반에 쓰임

---

## 텍스쳐 (texture)
Vector를 **텍스처 좌표계(특히 UV)**로 해석

- 2D 중심 (주로 X/Y만 의미)
- UV 기반 패턴 이동/회전/스케일

### 사용
- UV 좌표를 직접 다루는 경우
- 이미지 텍스처의 타일링/회전

---

## 방향 (vector)
입력 벡터를 **방향(direction)**으로 해석

- 벡터가 어느 방향을 향하나
- Location 무시
- Rotation / Scale만 적용

### 사용
- 바람 방향
- 왜곡 방향
- 벡터 기반 변형

---

### 표면의 법선 (normal)
입력 Vector를 **표면 법선(normal)**으로 해석

- 길이가 1인 방향 벡터
- Location 무시

### 사용
- Normal Map 방향 회전
- Bump/Normal 결과 정렬

---

## 입력 소켓
## Location (이동)
패턴을 공간에서 이동 시킴

- Location X/Y/Z 값을 바꾸면 → 노이즈가 흘러가는 것처럼 이동

### 사용
- 패턴 반복 방지
- 애니메이션(시간에 따라 Location 이동)

---

## Rotation (회전)
좌표계를 회전시켜 패턴 방향 변경

### 사용
- 줄무늬/방향성 패턴
- 스크래치 방향 정렬

---

## Scale (스케일)
좌표 공간을 늘리거나 줄임

- Scale ↑ : 패턴이 촘촘해짐
- Scale ↓ : 패턴이 커짐

| 항목  | Mapping Scale | Noise Scale |
| --- | ------------- | ----------- |
| 위치  | 좌표 변환         | 패턴 내부       |
| 영향  | 모든 패턴         | 해당 Noise만   |
| 일관성 | 높음            | 국소적         |
| 용도  | 전체 크기 통제      | 패턴 성격       |

---

# Color Ramp (value remapper)
0~1 값(입력)을 **사용자가 정의한 구간과 규칙으로 재매핑 하는 노드**

- **입력**: 스칼라 값 (대부분 Noise의 Fac)
- **출력**: Color(RGB) 또는 사실상 값(마스크)

---

## Color Mode
색 보간하는 공간을 바꾸는 옵션

## RGB (기본 값)
빨강/초록/파랑 각 채널을 독립적으로 보간

- 중간색이 회색/탁해지기 쉬움
- 채널이 섞이면서 채도가 줄어듦

### 사용처
- 마스크 (흑/백)
- Roughness / Height 같은 값 용도
- 색이 아니라 수치로 쓰일 때

---

## HSV (Hue / Saturation / Value)
색상(Hue)을 원형으로 보간. 채도(S), 명도(V)를 따로 처리

- 색이 무지개처럼 회전
- 채도 유지가 잘 됨
- 중간이 회색으로 죽지 않음

### 사용처
- 색 변화가 돌아가는 연출
- Thin Film, 스타일라이즈드 컬러

---

## HSL (Hue / Saturation / Lightness)
HSV와 유사하지만 명도를 **Lightness(밝고 어두움의 균형)**로 처리

- HSV보다 톤이 안정적
- 과도한 밝기 튐이 적음
- 색 변화가 더 페인트 같은 느낌

### 사용처
- 자연 재질의 색 변화
- 채도는 유지하되 과장되지 않게
- 현실적인 색 변화는 HSL이 유리

---

## 정리

| 컬러 모드  | 색 변화  | 채도 유지 | 밝기 안정성 | 주 용도   |
| --- | ----- | ----- | ------ | ------ |
| RGB | 채널 혼합 | 낮음    | 중간     | 마스크/값  |
| HSV | 색상 회전 | 높음    | 낮음     | 효과/스타일 |
| HSL | 색상 회전 | 중간    | 높음     | 자연 색   |

---

## Interpolation Type
보간 옵션

| 보간  타입    | 특징      | 대비    | 용도      |
| ------------ | ------- | ----- | ------- |
| Constant     | 값이 계단식. 중간값 없음 | 매우 강함 | 하드 마스크  |
| Linear       | 직선 보간     | 중간    | 범용      |
| Ease         | S-curve (부드러운 시작과 끝), 자연스러운 경계값 | 낮음    | 자연 전이   |
| Cardinal | Linear보다 부드럽고 B-Spline보다 대비 유지  | 중~높음  | 자연 + 선명 |
| B-Spline     | 둥글고 부드러운 곡선. 대비 감소    | 낮음    | 연기/피부   |

---

## ColorRamp vs Map Range
보이면 ColorRamp, 숫자면 Map Range

| 항목    | ColorRamp     | Map Range        |
| ----- | ------------- | ---------------- |
| 목적    | 시각적 재해석       | 수치적 재매핑          |
| 출력    | Color(또는 마스크) | 값                |
| 조절 방식 | 슬라이더/곡선       | 숫자               |
| 적합 용도 | 마스크/색         | Roughness/Height |

# color mix
컬러 믹스 노드 블렌드 타입 옵션 정리

| 옵션(Blend Type)                   | 결과 R (채널별, 0~1 가정)                      | 하는 일(로직)                           | 절차 텍스처에서 흔한 용도                                     | 주의/특징                                  |
| -------------------------------- | --------------------------------------- | ---------------------------------- | -------------------------------------------------- | -------------------------------------- |
| Mix                              | `R = B` (그리고 최종은 `lerp(A,B,f)`)         | 단순 선형 보간                           | 마스크로 두 패턴 섞기                                       | 가장 “중립적”                               |
| Add (Linear Dodge)               | `R = A + B`                             | 밝기(에너지) 누적                         | 디테일 “더하기”, 하이라이트/먼지 얹기                             | 클램프 없으면 1 초과 가능(UE에서는 톤매핑/클램프 주의)      |
| Subtract                         | `R = A - B`                             | 밝기 제거                              | 패턴 깎기, 골/홈 만들기                                     | 음수 가능(클램프 여부 확인)                       |
| Multiply                         | `R = A * B`                             | 겹치는 곳만 남기며 전체 어두워짐                 | 마스크 곱, AO 느낌, 색/러프니스 변조                            | 0에 민감(검정이 강하게 먹음)                      |
| Divide                           | `R = A / max(B,ε)`                      | B가 작을수록 밝아짐                        | 특정 영역만 강조/보정(드물게)                                  | 0 근처 폭주(노이즈 망가질 수 있음)                  |
| Darken                           | `R = min(A, B)`                         | 둘 중 더 어두운 값 선택(= “최솟값 합성”)         | **하이트/마스크에서 ‘깎기’**, 균열/홈 누적                        | 절차 모델링에 매우 유용(형태가 단단해짐)                |
| Lighten                          | `R = max(A, B)`                         | 둘 중 더 밝은 값 선택(= “최댓값 합성”)          | **하이트/마스크에서 ‘쌓기’**, 돌출 누적                          | 형태 합성에 좋음                              |
| Screen                           | `R = 1 - (1-A)(1-B)`                    | Multiply의 반대 성향(밝게)                | 먼지/안개/밝은 얼룩 얹기                                     | 흰색에 민감(흰색이 강하게 먹음)                     |
| Overlay                          | `A` 기준: 어두운 영역은 Multiply, 밝은 영역은 Screen | 대비 증가(중간은 유지, 극단은 강화)              | 바위 표면 대비/얼룩 강화                                     | “Base가 기준”이라 입력 순서가 중요                 |
| Hard Light                       | `B` 기준 Overlay(역할 교대)                   | 대비 증가(Overlay와 비슷)                 | 마스크/디테일을 더 공격적으로                                   | 입력 순서 영향 큼                             |
| Soft Light                       | Overlay보다 완만한 대비 증가                     | 부드러운 콘트라스트/조명 느낌                   | 자연스러운 얼룩/톤 변화                                      | 구현식이 복잡(버전에 따라 미세 차이)                  |
| Difference                       | `R = abs(A - B)`                        | 차이만 남김                             | 에지/변화 강조, 패턴 비교                                    | 의도 안 하면 지저분해질 수 있음                     |
| Exclusion                        | `R = A + B - 2AB`                       | Difference보다 약한 반전/차이              | 은은한 변조                                             | 색이 “뜬” 느낌 가능                           |
| Color Dodge                      | `R = A / (1-B)`                         | B가 밝을수록 급격히 밝아짐                    | 번짐/하이라이트, 금속성 광택 흉내                                | 폭주 쉬움(클램프/밸런스 필수)                      |
| Color Burn                       | `R = 1 - (1-A)/B`                       | B가 어두울수록 급격히 어두워짐                  | 그을림/때/침식                                           | 0 근처 폭주 주의                             |
| Linear Burn                      | `R = A + B - 1`                         | Add에서 1만큼 빼서 어두워짐                  | 하이트 “깎기”를 더 강하게                                    | 음수 가능                                  |
| Linear Light                 | `R = A + 2B - 1`                    | B를 “0.5 기준으로” A에 더하거나 빼서 대비/변형 | 마스크/하이트를 ‘중간 회색 기준’으로 밀어 대비를 키움, 디테일을 공격적으로 주입 | 값이 쉽게 0~1 밖으로 튐 → 클램프/강도 조절 중요         |
| Vivid Light                      | Dodge/Burn을 더 강하게 혼합                    | 매우 공격적인 대비/광택                      | 특수효과성 마스크 강화                                       | 폭주/밴딩 위험 큼                             |
| Pin Light                        | 밝/어두움 중 일부만 교체                          | 톤을 “잘라” 교체                         | 마스크를 더 조각내기                                        | 쉽게 부자연스러움                              |
| Hue / Saturation / Color / Value | 색상 성분만 교체                               | 색 공간 기반 합성                         | 알베도 색 교정/분리                                        | **데이터 맵(roughness/height)에 쓰면 의미가 깨짐** |
